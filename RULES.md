# Project Rules

1. Work in small slices.
2. Preserve the AI Trading College and BIGBoss Trader OS focus for V1.
3. Treat the shared memory layer as core product infrastructure, not an afterthought.
4. Do not build live trading or order execution in V1.
5. Use human approval gates for high-risk trading, product, financial, and publishing decisions.
6. Keep agent responsibilities separate and testable.
7. Update `TASKS.md` and `CHANGELOG.md` after project changes.
8. Do not overwrite the referenced dashboard example unless the user explicitly asks.
